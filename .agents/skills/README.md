# Repo Skills

These files are repo-local guidance for coding agents working in `local-expo-mcp`.

Use them as follows:
- [repo-conventions.md](repo-conventions.md): high-level implementation rules and architecture defaults
- [folder-structure.md](folder-structure.md): where new code, tests, fixtures, and docs belong
- [public-contracts.md](public-contracts.md): MCP surface stability rules
- [testing-and-validation.md](testing-and-validation.md): TDD and required validation flow
- [release-and-publishing.md](release-and-publishing.md): commitlint, semantic-release, and publishing policy
- [docs-and-guidance.md](docs-and-guidance.md): README vs AGENT vs plan ownership

Keep the public package documentation in [README.md](../../README.md). Keep implementation guidance here and in [AGENT.md](../../AGENT.md).
